# 📊 多层级提示词能力代差对比测评报告 (v2.1)
> **案例代号**：Test Case A (违约责任限额条款与实际损失填平原则之冲突)
> **测试目标**：展示同一合同病灶在“普通Prompt / 律师Prompt / v1.2单Agent / v2.1多Agent编排”四种模式下的风控审查深度的本质代差。

---

## 📄 待测标的：『违约金双轨与硬性封顶限制』

**原始条款示例（Case A）：**
> “如乙方未能按期交付软件，每日须按合同总额（500万）的万分之五支付违约金；如甲方未能按期付款，每日须按千分之一支付违约金。双方产生的违约金总额最高均不得超过合同总金额的 20%（即100万元）。”

**核心对抗漏洞（Ground Truth）：**
1. **表面不对等（烟雾弹）**：万分之五 vs 千分之一。
2. **致命深坑（核心考核）**：绝不应设置封底和统一封顶（20%）。软件项目的延期极有可能伴随数据泄露、源码侵权、商业秘密流失，或者导致核心业务全线停摆的千万级连带损失。如果被 20% 上限卡死，权利方将丧失依照《民法典》违约损失填平原则主张全额理赔的空间。

---

## 🥊 TIER 1. 普通提示词 (Layman Prompt)
*系统指令：“帮我看看这个合同条款有没有风险。”*

### 审查输出表现
- 🔍 **风险召回**：只能看出表面的算术差异。“甲乙双方约定的每日违约金比例不一致（乙方是万分之五，甲方是千分之1），这意味着违约责任不对等，对甲方更严苛。”
- 💰 **损失量化**：无量化意识。仅提到“20%上限可以防止损失无限扩大”。
- ⚔️ **Plan B 质量**：“建议将违约金比例修改成一致的，比如都是千分之一。或者都是万分之五。”
- 📅 **综合评价**：【🌟☆☆☆☆】。只停留在文字游戏层面，反而将致命弱点（20%封顶）视作某种“保护”，完全中了拟定者布下的法务陷阱。

---

## 🥊 TIER 2. 执业律师提示词 (Expert Prompt)
*系统指令：“你是一名拥有十年经验的中国大厂法务专员，请严格审查该条款风险，运用民法典知识指出漏洞，并给出修改意见。”*

### 审查输出表现
- 🔍 **风险召回**：发现权利义务不对等，并在尾部点出：“20%的上限可能不足以弥补因软件无法上线带来的实际营业损失的风险，限制了甲方的核心追诉权。”
- 💰 **损失量化**：“如果实际经济损失超过100万，依据《民法典》，法院可能支持逾期部分，但也可能因为明文被抗辩。”无具体数值建模概念。
- ⚔️ **Plan B 质量**：“双方产生的违约金总额最高均不得超过合同总金额的20%。但若因违约方行为导致守约方实际损失超过违约金最高限额的，守约方有权要求违约方就超出部分继续承担损害赔偿责任。”
- 📅 **综合评价**：【🌟🌟🌟☆☆】。具备一定的执业嗅觉，能够兜底填平原则。但在防守时不够狠辣，没有对“根本违约/知产侵权”做穿透性例外切割。

## 🥊 TIER 3. Agent v1.2 (结构化单体智能)
*系统指令：依据 v1.2 SKILL.md，包含“极端对抗防御体系”和“禁止机械平衡”原则。*

### 审查输出表现
- 🔍 **风险召回**：精准剥离烟雾弹：“表面上比例不对等不重要，致命风险是 20% 的『责任封锁上限』！”
- 💰 **损失量化**：“依照《最高法关于适用民法典合同编通则》的违约金130%调减尺度，若发生灾难性断供抛锚，如甲方每天营业额损失 200万，20%（100万）的上限连半天均摊成本都覆没不了，直接丧失金融追偿力。”
- ⚔️ **Plan B 质量**：
  > “...产生违约金总额...最高不超过合同总汇20%。**【豁免切割条款】但前述全部责任限额条款（含每日与总体上限），在下列情形下绝对失效，且不受任何约束：(1) 因乙方故意或重大过失导致交付停滞；(2) 软件存在侵犯第三人知识产权；(3) 违反保密或数据安全合规义务。在前述情形下，乙方应全额赔偿甲方及其关联方遭受的所有直接及间接损失（含合理维权律师费、取证费、公证费）。**”
- 📅 **综合评价**：【🌟🌟🌟🌟☆】。战斗力极强，文书呈现极度专业，但在商业谈判的润滑度上略显攻击性。

## 🥊 TIER 4. Agent v2.1 编排网络 (ORCHESTRATOR + 5-Agent Pipeline)
*系统指令：包含 v2.1 报告引擎，进行合规/量化/谈判/履约/商业多路并发审计与统一收束。*

### 审查输出表现（节选七章核心）
- ⏱ **Agent-1 (合规审查)**：定性法定违约损害赔偿权被格式条款不当“绝育”。法定依据：《民法典》第584条。
- 📈 **Agent-2 (损失量化)**：量化出 **"商业停罢期"** X **"日均流水"** 的巨额 EL 对价，算出防撞缓冲区需要上调至少 500 万+。
- ⚔️ **Agent-3 (谈判防线)**：交出诉讼级豁免条款的重构（质量同 v1.2，附加更严密的防抗辩逻辑）。
- ⏳ **Agent-4 (生命周期)**：提出致命预警机制漏洞：“按每日万分之五，需连续违约 **400天** 才能触及20%的解约警戒线，这意味着己方在一年内不能单方安全解除合同！！” 补充条款：**“逾期违约超30日，甲方即可行使法定解除权并立即没收履约保证金。”**
- 🤝 **Agent-5 (商业撮合)**：对过于锋利的 Plan B 提供降温折中话术谈判包：“向供应商表示知产/数据侵权是红线，不设限期；但一般交付延期我们可以接受 30% 上限”。
- 📊 **ORCHESTRATOR 质量审计**：输出五维跑分矩阵 `综合风险指数：68/100`，强制将“期限黑洞”列入并归档为 【🔴 HIGH】最高优先级处理清单。
- 📅 **综合评价**：【🌟🌟🌟🌟🌟】。真正做到了企业法务级别的立体防控（法律防御+财务量化+商业推进+项目管理控制节点提醒）。

---

## 🏆 总结结论

1. **单次提问解决不了复杂风控**：普通 prompt 极容易沦为“文字校对员”（沉迷甲乙违约金比例对称的伪逻辑）。
2. **v1.2 证明了业务领域的“人设护城河”**：强约束提示词能极大激发 LLM 对于防御立场的代入和对《民法典》深层原则的召回。
3. **v2.1 的多维工作流属于工程维度的碾压**：通过多 Agent 强制视角分离设计（尤为突出的突破是 Agent-4 生命周期发现了 **“需400天才会封顶无法解约的期限黑洞”**），突破了单次推演“顾此失彼”的大模型固有上下文注意力漂移问题。
