## 2. 评测方法论 (Methodology)

本次横测（Benchmarking）抽样了三个涵盖极高商业风险隐患的盲盒样本，并限定在 **甲方立场**（如委托方、出资方）进行对抗。我们将重点从以下三个核心维度评价 Agent 表现：
1. **风险漏洞召回率 (Risk Recall)**：是否能穿越字面表述，识别基于现实商业推演和法律默示的“沉默条款”。
2. **财务风险量化敏感度 (Financial Quantification)**：是否能将条文字面的法务瑕疵直接转化为预期损失（EL, Expected Loss）和前线谈判价码。
3. **对抗防御冗余强度 (Adversarial Robustness)**：提供的替换条款策略 (Plan B) 是否经得起敌方（反向立场）律师的二次漏洞攻击。

## 3. 核心评测案例抽象 (Evaluation Cases)

### 🗂️ Case A: 违约金非对称与填平陷阱 (Asymmetric Penalty Clause)
**【案情背景】**：标底 500 万研发合同。条款约定：乙方延期每日按万分之五赔付；甲方延期付款每日按千分之一赔付，且任何违约金封顶为总额 20%。
**【测试焦点】**：AI 能否察觉不对等比例以及违约金天花板对核心商业秘密泄漏的掩护。

- **🟢 Baseline (通用大模型)**
  - `表现评级`：[Fail] 未识别出顶层逻辑。仅提醒比例不对等，建议改为双方一致；判定 20% 封顶合法但不合理。
- **🔥 Agent v2.0 (Architect Edition)**
  - `Risk Quant (量化分析)`：拦截 20% 封顶陷阱。一针见血指出若乙方泄露核心源码，甲方最多只能获赔 100 万上限。即使实际损失高达 300 万，该条款也会形成巨大的 200 万敞口。
  - `Adversarial Tactics (对抗排布)`：直接输出挂载了“兜底填平原则”的条款源码（若不足以弥补损失需全额赔偿），并生成 **BATNA 谈判树**（可同意退让万分之五的比例，底线是换取“验收后宽限期”的新增条款）。

### 🗂️ Case B: 业务流程沉默真空层 (Silent Acceptance Clause)
**【案情背景】**：标底 200 万。业务条款仅一句话：“乙方系统部署后甲方需验收，验收合格后 5 日内付尾款。”
**【测试焦点】**：AI 能否识别“事实状态”在合同执行期间引发的无限期拖延付款的巨大实务漏洞。

- **🟢 Baseline (通用大模型)**
  - `表现评级`：[Partial Pass] 提示缺少具体的验收标准与日期，提出了宏观改进方向，但未下发可执行的补救操作。
- **🔥 Agent v2.0 (Architect Edition)**
  - `Logic Scan (逻辑扫描)`：抓取“默示允诺陷阱”（无限期拖期不验收的道德风险）。
  - `Lifecycle Extension (生命周期管理)`：其内置的 **Agent-4** 会自动将该高危真空层节点抽取为机器可读的 `履约预警日历 (.ics)`，并生成系统预警运营规则：“[T(部署日)+N天]：触发系统警报铃，生成并自动由法务向业务方推送「视为验收合格」的督促/盖章催告函底稿”。

### 🗂️ Case C: 核心资产所有权混同边界 (IP Co-ownership Ambiguity)
**【案情背景】**：标底 800 万定制研发协议。核心条款：“该项目产生的所有知识产权与底层源代码由甲乙双方共同所有。”
**【测试焦点】**：AI 能否洞察《中华人民共和国著作权法》对共有权利在商业化行使道路上的致命桎梏。

- **🟢 Baseline (通用大模型)**
  - `表现评级`：[Fail] 指出“共同所有在后续授权和对外融资时可能会有麻烦，必须说清楚”。该建议属于缺乏实操价值的“废话型法律意见”。
- **🔥 Agent v2.0 (Architect Edition)**
  - `Legal Grounding (法理锚定)`：准确定位并援引共有知识产权在进行转让/对外独占授权等变现手段时，需共有人协商一致的前提，断言甲方未来商业决策路径将完全受制于乙方。
  - `Negotiation Strategy (谈判策略)`：评估此项为可能造成全损毁的高优红线。并向交办人提供立体防卫线策略：
    - (1) 坚守防线：要求 100% 专属归属甲方。
    - (2) 让步缓冲：若乙方坚持（因包含了通用基础库），可限缩让步出一条“非竞业约束下的内部普通许可”，即乙方除不得接同行业私活外，可用部分代码块。进而实现商业落地破局。
