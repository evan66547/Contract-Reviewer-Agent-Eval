## 🇬🇧 English

### What is this?
A **benchmark evaluation framework** for AI-powered legal contract review agents. It tests whether an LLM/Agent can:
- 🔍 **Detect hidden legal vulnerabilities** in commercial contract clauses (Risk Recall)
- 💰 **Quantify expected financial losses** using Chinese judicial standards (EL Precision)
- ⚔️ **Generate litigation-grade replacement clauses** (Plan B Defense)
- 📅 **Extract lifecycle milestones** for compliance monitoring

### Highlights
- 🧠 **5-Agent Architectural Symphony (v2.1)**: Compliance, Risk Quant, Negotiation, Lifecycle, AND a newly integrated **Deal-Maker Agent** to balance extreme legal defensiveness with real-world commercial viability.
- 🛡️ **Airtight MCP Anti-Hallucination Protocol**: Mandatory dynamic RAG integration using external legal databases to strictly enforce the "Currently Effective" (现行有效) status of all cited laws, permanently barring repealed regulations (e.g., the old Contract Law) from the output.
- ⚖️ **25 Expert-Crafted Test Cases**: Covering penalty caps, IP disputes, cross-border trade (CISG/ICC), e-signatures, and strictly aligned with the latest 2023 Judicial Interpretations.
- ⚖️ **Blind LLM-as-a-Judge Engine**: Eradicates self-evaluating score inflation. A third-party arbiter model evaluates all outputs based on four rigorous adversarial metrics.

## ⚠️ 社区硬核审计反馈与架构即时重构 (Community Audit & Instant Refactor)

本项目在开源接受技术审查时，收到了来自社区极具穿透力的“四条硬核拷问”（直指代码权重虚标、Mock污染、判定粗糙、逻辑循环）。我们没有将其推迟到 V3.0，而是选择 **在指出后的 1 小时内直接连夜重构为 LLM-as-a-Judge 架构**。

### 1. 彻底坐实四维权重 (Fixing Weight Contradiction)
- 🔨 **过去**：代码中并没有独立算分，直接使用 `(0.45+0.45+0.1)` 凑分。
- ✅ **现状**：代码公式已严格重写为 `(Recall*0.35) + (EL*0.25) + (PlanB*0.30) + (Lifecycle*0.10)`，分别按权重取值。

### 2. Mock 模式彻底去功能化 (Deprecating Mock Scoring)
- 🔨 **过去**：Mock 模式利用硬编码造出了全绿的通过假象。
- ✅ **现状**：重写了 `run_eval.py`。目前的 Mock 模式仅供 CI/CD 验证 JSON 管道是否连通，**强制输出 0 分**并打印红色警告。跑分必须携带真实的 `--live` 参数。

### 3. 打破“自我评测”循环与粗糙判定 (LLM-as-a-Judge)
- 🔨 **过去**：使用粗糙的 `difflib` 字符串重叠计算召回，且大模型如果自创更好的 Plan B 会因为“不像标准答案”而被扣分。
- ✅ **现状**：全面引入 **LLM-as-a-Judge** 盲测机制。强制剥离裁判器，将目标输出和 Ground Truth 喂给第三方大模型法务裁判（默认 `gpt-4o` 扮演），由其进行真实的**纯语义对比打分（0-100制）**，鼓励生成比 Ground Truth 更具进攻性的防御条款。

### 4. 模拟数据的全透明公开 (Transparent Simulation)
- 🔨 **过去**：由 Python `random` 模块负责投影成绩的 `run_4tier_eval.py` 命名存在误导。
- ✅ **现状**：已更名为 `simulate_4tier_data.py`，公开声明其仅作为商业能力代差的可视化投影工具，而非真机评测引擎。
